# Sprint 17 Spec — 数据安全加固与企业准备

## 目标

补上 `plan-v2` 里仍未落地的生产级隐私安全项，为后续企业化部署做准备。

## 背景

当前系统已经实现脱敏，但还没有真正做到字段级加密、清晰的数据边界与更严格的安全收口。这部分不影响 MVP 使用，但会影响生产可交付性。

## 本 Sprint 范围

### 1. 字段级加密

**做**：
- 识别高敏字段：
  - message content
  - episodic memory content
  - summary
  - trace 中的敏感片段
- 制定并实现加密策略

### 2. 数据边界文档化

**做**：
- 明确哪些数据：
  - 原文存储
  - 脱敏存储
  - 可进入评估
  - 可进入管理端统计

### 3. 访问与保留策略

**做**：
- 形成最小数据保留与删除策略
- 明确评估样本、trace、session summary 的生命周期

### 4. 安全回归

**做**：
- 验证加密不影响现有会话、记忆、trace、评估流程

## 不在范围

- 完整企业 SSO
- 多租户权限系统
- 法务与合规文本本身
