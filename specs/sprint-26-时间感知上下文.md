# Sprint 26 Spec — 时间感知上下文

## 目标

为心雀的跨天对话、干预跟进和作业回访补齐“时间感知上下文”能力。

本 Sprint 的目标是让系统在把历史事件、最近干预、未完成作业和会话摘要注入上下文时，不只带内容，还要带：

- 当前时间
- 当前时区
- 历史条目的绝对时间
- 历史条目的相对时间

从而让 LLM 能区分“今天”“昨天”“几天前”“很久以前”，更准确地理解 follow-up 场景。

## 背景

当前系统已经具备：

- `recall_context()`
- `search_memory()`
- `SessionState / SessionStateHistory`
- `layered context`
- 干预历史与作业跟进

但当前时间语义仍然不完整：

- `recall_context()` 只返回部分日期字段
- 运行时没有把“当前时间/时区”明确注入给模型
- `session_context.py` 的核心上下文卡片没有保留关键时间信息
- 历史事件有时间字段，但模型无法稳定比较“它和现在是什么关系”

因此，需要一个独立 Sprint，把时间信息从“数据库元数据”升级为“模型可用上下文”。

## 本 Sprint 范围

### 1. 注入当前时间与时区

**做**：

- 在运行时生成统一的 `runtime_time_context`
- 至少包含：
  - `current_time_iso`
  - `current_date`
  - `timezone`
- 让这部分信息进入心雀每轮的核心上下文

**验收要点**：

- 模型不再需要猜“今天是几号”

### 2. 为历史条目补齐绝对时间和相对时间

**做**：

- 为 `recent_interventions`、`pending_homework`、`last_session_summary`、相关记忆补：
  - 绝对时间（ISO 或日期）
  - 相对时间标签（如“今天”“昨天”“3天前”）
- 时间粒度不足时，明确标记为日期级时间

**验收要点**：

- 干预和作业的新鲜度能被上下文直接表达

### 3. 让时间信息进入 layered context

**做**：

- `session_context.py` 的 `stable_state` 保留关键时间字段
- `render_layered_context_message()` 在核心上下文卡片中体现时间信息
- 保证时间信息不是只存在 Tool 返回 JSON 中，而是真正进入每轮主上下文

**验收要点**：

- 长会话核心上下文对时间有感知

### 4. 保持现有主链路兼容

**做**：

- 不改变 chat 主返回协议
- 不改变历史记录或 trace 的主结构
- 以增量字段和上下文增强为主

## 验收标准

- 运行时已注入当前时间和时区
- 历史条目已具备绝对时间和相对时间
- 时间信息已进入 layered context 的核心上下文卡片
- chat 主链路保持兼容
- 至少存在自动化测试或可重复验证覆盖本次改造
