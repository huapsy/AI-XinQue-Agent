# Sprint 39 - 外部向量索引抽象与检索升级

## 背景

当前心雀已经具备稳定 embedding、本地混合检索与情景记忆读写主链路，足以支撑 MVP。但从 `plan-v2.1` 的目标态看，当前实现仍有两个问题：

- 检索层与本地实现耦合较强，不利于后续切到外部索引
- 缺少更明确的索引同步、失败回退和质量评估边界

因此，本轮不应简单理解为“换一个向量库”，而应先把检索层从产品逻辑中抽象出来。

## 目标

本轮 sprint 的目标，是建立可切换的检索抽象层，并完成外部向量索引接入所需的最小同步与回退设计。

完成后应达到：

- 本地检索与外部索引有清晰接口边界
- 记忆写入、更新、删除的索引同步有明确策略
- 外部索引失败时存在可验证回退路径
- 检索质量可以通过回归样本进行比较

## 范围

### 1. 检索抽象层

把当前 memory retrieval 从具体实现中抽出，至少区分：

- embedding 生成
- 记忆主存储
- 向量检索执行
- 结果融合与排序

### 2. 外部索引接入最小版

接入一条外部索引路径，但本轮重点是：

- 抽象正确
- 同步链路稳定
- 回退逻辑清晰

而不是追求复杂的供应商特性。

### 3. 检索质量回归样本

建立最小检索回归样本，覆盖：

- 正确召回近期关键事件
- 不因相似文本噪声误召回
- 回访场景下能优先命中高价值历史

### 4. 运行与配置文档

补齐外部索引启用条件、降级策略、运维注意事项。

## 非目标

- 本轮不把数据库整体迁移到 PostgreSQL
- 不做复杂 ANN 参数调优竞赛
- 不在本轮引入大规模离线回填系统

## 预期结果

如果本轮完成，心雀的情景记忆检索将从“本地可用实现”推进到“具备外部索引演进路径的稳定架构”。
