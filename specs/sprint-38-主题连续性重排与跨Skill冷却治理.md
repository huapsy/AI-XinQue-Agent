# Sprint 38 - 主题连续性重排与跨Skill冷却治理

## 背景

`plan-v2.1` 明确要求心雀具备长会话、多轮回访与干预后跟进能力。当前项目已经有：

- 时间感知上下文
- recent intervention follow-up guardrail
- 多干预优先级最小版
- state / timeline 最小持久化

但在实际产品形态上，仍有两个明显缺口：

- 用户跨轮回访时，心雀对“当前最该延续的主线”把握还不够强
- 多个 skill 在近期交替出现时，缺少更稳定的冷却治理，容易出现重复推荐或切换过快

## 目标

本轮 sprint 的目标，是提升长程对话中的主线连续性，让心雀更稳定地做到：

- 抓住最近仍然重要的主题
- 合理跟进近期干预与作业
- 避免在短时间内重复推同类 skill
- 在继续探索、继续跟进、切换新主题之间做更一致的排序

## 范围

### 1. 主题连续性重排

新增或补强对以下信息的联合排序：

- 当前 session state
- 最近 phase timeline
- 最近 intervention / homework
- 检索到的 episodic memory
- 当前用户消息中的新鲜信号

目标是形成一个“本轮最值得继续的主线”排序结果，而不是只看单一命中分数。

### 2. 跨 Skill 冷却治理

把当前最小版冷却逻辑升级为更稳定的治理规则，至少覆盖：

- 同 skill 的冷却窗口
- 同类别 skill 的冷却窗口
- 刚完成的干预优先跟进，而不是立刻推荐新练习
- 用户明确表达不喜欢或无帮助的 skill，在短期内降低优先级

### 3. 主题切换解释性

当系统从“继续旧主线”转向“进入新主线”时，内部状态应具备最小解释性，方便：

- trace 分析
- evaluator 判断
- 后续运营面板读取

### 4. 回归测试

补齐主题排序、冷却、近期干预跟进相关测试，避免长会话回归。

## 非目标

- 本轮不扩大量新 skill 内容
- 不引入外部向量数据库
- 不大改前端聊天交互

## 预期结果

如果本轮完成，心雀在回访和长会话中的“主线感”会明显更强，也更符合 `plan-v2.1` 对关系连续性和干预节奏的要求。
