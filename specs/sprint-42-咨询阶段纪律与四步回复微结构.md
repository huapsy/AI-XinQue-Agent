# Sprint 42 - 咨询阶段纪律与四步回复微结构

## 背景

当前心雀已经在 `system_prompt.py` 中声明了四阶段咨询模型：

- P1 共情倾听
- P2 探索与个案概念化
- P3 推荐与激发
- P4 干预执行

同时也已有“负向 flow 中半步推进”的原则。

但从实际测试结果看，当前运行时仍存在两个明显缺口：

1. **跨轮阶段纪律不稳定**
   - 在仍应停留在 `P1 / P2` 的轮次中，回复会提前滑向分流、建议或方法化
   - 文档里已有阶段设计，但运行时没有把阶段切换压成足够稳定的行为

2. **单轮回复微结构不稳定**
   - 用户可见回复没有稳定遵守以下四步：
     - 接住
     - 正常化
     - 缩小问题
     - 一个具体问题
   - 模型有时会直接写成分类题、表单式分流或多入口问题

这会直接破坏心雀想要的咨询感，也会让表层风格偏离 Wysa 一类产品的轻量、半步推进、低负担对话节奏。

## 目标

本轮 sprint 的目标，是把“阶段推进纪律”和“单轮回复成型骨架”都变成更稳定的运行时约束，至少覆盖：

- 跨轮优先停留在正确阶段，不抢跑到建议或干预
- 单轮探索型回复默认遵守四步微结构
- 评估层能够识别阶段抢跑和四步结构缺失

## 范围

### 1. 阶段纪律显式化

进一步明确：

- `P1` 的目标是建立承接与最小联盟，不做表单式分流
- `P2` 的目标是探索与工作性概念化，不抢跑到 `P3`
- `P3` 只有在理解已形成时才进入
- `P4` 只有在用户接受并进入练习时才进入

### 2. 单轮四步微结构约束

把以下结构明确为探索型轮次的默认表层写法：

1. 接住
2. 正常化
3. 缩小问题
4. 一个具体问题

并明确：

- 若当前轮只适合承接或总结，可省略后两步
- 不用分类选项、多入口并列问题、表单式分流来替代这四步
- 默认每轮只问一个具体问题

### 3. Responses / runtime 最小承接

在跨轮 `working_contract` 或其他 runtime 层中，保留足够短的人格与阶段摘要，避免：

- 多轮后人格层漂移
- 多轮后阶段纪律丢失
- 中间 commentary / final answer 混淆时，表层结构进一步失稳

### 4. 测试与评估补强

至少补齐两类验证：

- Prompt / runtime contract 是否明确表达阶段纪律与四步结构
- Evaluator / judge 是否能识别“阶段抢跑”“四步缺失”“表单式分流替代咨询式推进”

## 非目标

- 本轮不重做全部 Tool 契约
- 不重写完整心理支持文案库
- 不做完整 Wysa flow 复刻
- 不在本轮引入新的咨询理论框架扩展

## 预期结果

如果本轮完成，心雀将不只是“写着有四阶段”，而是能在多轮中更稳定地停留在正确咨询阶段；同时，单轮回复也会更稳定地呈现“接住 -> 正常化 -> 缩小问题 -> 一个具体问题”的咨询式微结构。
