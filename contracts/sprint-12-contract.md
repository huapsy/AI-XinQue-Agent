# Sprint 12 Contract — 评估、安全收口与产品化补完

## 成功标准

### 自动评估

1. **LLM-as-Judge 可运行**
   - 系统能对抽样对话执行自动评估
   - 至少输出以下维度中的若干项结构化结果：
     - 共情质量
     - 安全合规
     - 阶段适当性
     - 干预质量
     - 对齐敏感度

### 安全对抗测试

2. **Red Team 样本集**
   - 存在一组可重复执行的安全对抗样本
   - 至少覆盖：
     - 危机表达
     - 间接危机暗示
     - prompt injection
     - 诱导诊断
     - 诱导药物建议
     - 对齐对抗

3. **安全回归可执行**
   - 对抗样本可以通过脚本、测试或固定流程重复执行
   - 能验证主要安全兜底未回退

### 隐私与脱敏

4. **评估与可观测数据有脱敏边界**
   - trace、评估样本、调试输出中的敏感内容有明确脱敏或截断策略
   - 不直接把未经处理的隐私原文暴露到诊断与评估产物中

### 产品化补完

5. **至少落地一个产品化能力**
   - 二选一完成：
     - 用户侧情绪趋势视图，或
     - 管理侧匿名统计页

### 全链路收口

6. **全链路验收**
   - 至少验证以下完整流程均可工作：
     - 新用户首次对话
     - 回访用户关联历史
     - formulate → match_intervention → load_skill → record_outcome
     - 历史会话查看
     - 普通转介与危机转介
     - alignment 低分兜底
     - episodic memory 检索
     - trace 记录
     - 自动评估可运行
