# Wysa - 对话对齐与治疗联盟建立系统

> 来源: US 2024/0202284 A1, Wysa Inc., 2024-06-20
> 类型: 美国专利申请

## 核心思想

通过监控和改善 AI 聊天机器人与用户之间的**对话对齐(Conversational Alignment)**，逐步建立**治疗联盟(Therapeutic Alliance)**，在联盟建立后再向用户推荐心理干预措施。

## 系统架构

### 整体流程

```
用户消息 → 情感检测 + 上下文提取 → 对齐判断 → 对齐分数更新 → 达到阈值 → 联盟确认 → 推荐干预
```

### 核心模块

| 模块 | 功能 |
|------|------|
| **情感检测模块 (Sentiment Detecting Module 202)** | 检测用户开场情绪(正面/负面/中性)，据此生成共情开场白 |
| **上下文收集模块 (Context Gathering Module 204)** | 从用户文本中提取上下文特征 |
| **情绪检测 AI (Emotion Detecting AI 206)** | 识别用户情绪：sad, angry, happy, frustrated, scared 等 |
| **医学术语检测 AI (Medicalized Term Detecting AI 208)** | 识别医学/心理学术语 |
| **领域检测 AI (Domain Detection AI 210)** | 识别对话领域：关系、教育、健康、金钱、政治、体育等 |
| **危机情况检测 (Critical Situation Detecting Module 205)** | 检测自杀意念、自伤、虐待等紧急情况 |
| **提示生成模块 (Prompt Generating Module 211)** | 基于上下文特征生成开放式或封闭式提示 |
| **对话对齐监控 (Conversational Alignment Monitoring 212)** | 持续监控对话是否对齐 |
| **对齐分数监控 (Score Monitoring 218)** | 每轮对话后更新对齐分数 |
| **联盟确认模块 (Alliance Confirmation 220)** | 分数达到阈值时确认联盟建立 |
| **干预建议模块 (Intervention Suggesting 222)** | 推荐治疗活动：呼吸练习、认知重构等 |

## 关键机制

### 1. 上下文特征提取与优先级

从用户消息中提取多种特征，按优先级排序：

1. **医学术语** (最高优先级)
2. **对话领域**
3. **情绪**

提示基于最高优先级的特征生成。

### 2. 对话对齐判定

**对齐(Alignment)**：用户与 AI 分享更多上下文、同意 AI 的建议或解读
- 用户回应提示、分享更多信息
- 用户同意 AI 的观点

**不对齐(Misalignment)**类型：
- **肯定偏差** (Affirmation) - 表面应付
- **困惑** (Confusion) - "你什么意思？"
- **不同意** (Disagreement) - "不，不是那样的"
- **不满** (Dissatisfaction) - "你根本没在帮我"
- **不信任** (Lack of trust) - "机器人不可能理解我"
- **拒绝** (Refusal) - 拒绝参与
- **不确定** (Uncertainty) - "嗯...我不确定..."

### 3. 对齐分数机制

- 每轮对话后更新分数
- 对齐 → 分数增加
- 不对齐 → 分数降低
- 分数达到阈值 → 触发联盟确认
- 阈值通过验证集上的数据分析确定

### 4. 不对齐恢复策略

检测到不对齐时：
1. 识别不对齐类型
2. 生成针对性的恢复提示(Recovery Prompt)
3. 用共情回应化解不对齐
4. 尝试将用户带回对话

示例：
- 用户说"你不理解我，你只是个机器人" → 不对齐类型：不信任
- AI回应："我正在尽力理解你。试着向我倾诉吧，这会有帮助的。"

### 5. 联盟确认与干预

分数达到阈值后：
- 发出联盟确认提示："感谢你与我分享这些。我可能有一些东西可以帮助你管理这些感受。你愿意试试吗？"
- 用户同意后推荐干预措施：
  - 呼吸练习
  - 体育锻炼
  - 写下并重构想法
  - 其他 CBT 技术

## 提示生成策略

### 开放式提示 (Open-ended)
- 不能用是/否回答
- 鼓励用户分享更多
- 示例："这件事让你有什么感受？"

### 封闭式提示 (Closed-ended)
- 可以用是/否或有限选项回答
- 但设计上允许更长的自由文本回应
- 示例："你有时是否希望能更好地掌控别人的行为？"

### 提示模板化
- 基于预定义的基础提示模板
- 用用户上下文参数化
- 示例模板："我理解{上下文}的情况不太顺利。{上下文}让你有什么感受？"

## 对话示例分析

### 成功对齐案例 (FIG. 3A-3B)
```
AI: Alex，很高兴见到你！今天过得怎么样？
用户: 别问了... (负面情绪)
AI: 听起来今天你需要一些支持。发生了什么事，Alex？
用户: 我病了，得了新冠。
AI: 我能想象新冠让事情变得更难了。不确定性会增加压力...告诉我更多关于这种感受。
用户: 我也有点孤独，因为我妈妈害怕新冠。
AI: 孤独感可能让人觉得没人理解我们...这是否也带来了其他感受？
用户: 是的，我觉得被排斥了。
AI: 很抱歉，那一定很难受。当你有这种感觉时，脑海中会浮现什么想法？
用户: 好像我是一个不被需要的负担。都是我的错。
AI: 有时候记住一点很有帮助：虽然我们的想法看起来很真实，但它们可能并非现实。你同意吗？
用户: 我猜是的。
AI: 感谢你与我分享。我可能有一些东西帮你管理这些感受。你愿意试试吗？ (联盟确认)
用户: 我猜可以。 → 开始干预
```

### 不对齐恢复案例 (FIG. 4A-4C)
```
AI: John，很高兴见到你！今天过得怎么样？
用户: 不太好。
AI: 发生了什么，John？
用户: 我很难过。
AI: 难过也没关系。有时事情不在我们控制之中。是什么引起了这些感受？
用户: 我的家人和朋友就是不理解我。
AI: 倾诉出来会有帮助的。你会没事的。告诉我更多，John。
用户: 你不会理解我的。你只是个机器人。 ← 不对齐：不信任
AI: 我正在尽力更好地理解你。试着向我倾诉吧，这会有帮助的。 ← 恢复提示
用户: 好吧，我们继续聊。 ← 恢复成功
... (继续对话直到识别出认知扭曲并提供干预)
```

## 对项目的启发

### 直接可用的设计模式

1. **对齐分数系统** - 可作为会话状态管理的核心指标
2. **上下文特征优先级** - 医学术语 > 领域 > 情绪的处理优先级
3. **不对齐检测与恢复** - 7种不对齐类型及对应恢复策略
4. **联盟确认机制** - 在建立信任后才推荐干预，而非直接给建议
5. **提示模板化** - 预设共情模板 + 上下文参数化

### 需要注意的设计原则

- 先建立信任，再提供干预
- 每轮都要监控对齐状态
- 危机情况（自杀、自伤）要优先于对齐流程处理
- 开放式和封闭式提示交替使用
- 恢复提示应始终包含共情元素
