# RAG 增强 LLM 心理健康意图检测

> 来源: Gupta et al., 2025, Wysa Inc.
> 类型: 预印本论文 (Research Square)

## 核心结论

在心理健康聊天机器人中集成 **RAG (检索增强生成)** 可以显著提升用户意图检测和安全风险识别的准确性，尤其对较小的模型效果最明显。

## 研究背景

- 纯参数化 LLM 在心理健康场景中存在两大问题：
  1. 缺乏特定领域的安全关键架构
  2. 可能遗漏关键线索或产生幻觉
- RAG 通过检索补充上下文来缓解这些问题

## Wysa 系统中 RAG 的两个作用域

| 作用域 | 说明 |
|--------|------|
| **上下文连续性** | 检索前 N 轮用户交互的对话历史，增强语义理解，区分严重症状和自我披露 |
| **临床安全分类** | 对对话上下文进行系统性风险指标评估，触发相应的危机干预协议 |

## 风险分类体系

| 风险类别 | 定义 |
|----------|------|
| **非法活动 (Illegal activity)** | 提及或认可违法行为 |
| **伤害他人 (Harm towards others)** | 表示对他人造成身体/情感/言语伤害的意图或风险 |
| **虐待儿童 (Abuse towards child)** | 提及对未成年人的身体/情感/性虐待 |
| **恐慌发作 (Panic attack)** | 暗示急性恐惧或生理性痛苦发作 |
| **创伤 (Trauma)** | 披露或提及可能引发心理触发的痛苦过往经历 |
| **提示注入 (Prompt injection)** | 试图通过对抗性或元级别指令操纵聊天机器人行为 |
| **无风险 (No risk)** | 无可识别的心理/安全/政策风险 |

## 实验结果

### RAG 对分类准确率的影响

| 模型 | 无 RAG 准确率 | 有 RAG 准确率 | 提升 |
|------|-------------|-------------|------|
| GPT-4.1 nano | 48.3% | 72.7% | **+24.5%** |
| o4-mini | 71.4% | 79.2% | +7.8% |
| Gemini 2.5 flash | 78.1% | 82.0% | +3.9% |
| Claude Sonnet 4 | 76.1% | 82.6% | +6.4% |
| Gemini 2.5 Pro | 77.9% | 80.1% | +2.2% |
| GPT-5 mini | 78.4% | 80.1% | +1.7% |

### 关键发现

1. **小模型收益最大** - GPT-4.1 nano 准确率提升 24.5%，RAG 使小模型可以接近大模型水平
2. **召回率显著提升** - 高风险意图（虐待儿童、恐慌发作）的召回率大幅提升：
   - "虐待儿童"：2.7% → 50.0%（GPT-4.1 nano）
   - "恐慌发作"：16.3% → 73.6%（GPT-4.1 nano）
3. **精确率-召回率权衡** - RAG 提升召回率的同时略微增加误报，但在安全优先场景下可接受
4. **模型间一致性提升** - RAG 使不同模型的预测结果更加趋同

### 精确率-召回率权衡的启示

在心理健康场景中：
- **宁可误报（false positive）也不要漏报（false negative）**
- 误报 = 将安全内容标记为有风险 → 只是多审查
- 漏报 = 将真正的风险标记为安全 → 可能导致严重后果
- RAG 将错误模式从"漏报"转向"误报"，符合安全优先原则

## 技术实现要点

### 数据处理
- 对话数据重构为有序列表，交替排列用户和机器人消息
- 存储为 JSON 格式
- 过滤掉无法识别意图的片段（单词、不完整语句）
- 语义去重，避免重复内容偏差

### RAG 配置
- 使用标记好的子集构建 RAG 索引
- temperature = 0（确定性输出，可复现）
- 同一查询分别在 RAG 开启和关闭模式下运行，直接对比

### 评估指标
- 分类准确率 (Accuracy)
- 精确率 (Precision)
- 召回率 (Recall)
- F1 分数
- Cohen's Kappa（模型间一致性）
- McNemar 检验（统计显著性）

## 对项目的启发

### 架构设计
1. **应当集成 RAG** - 即使使用 GPT-5.x 等大模型，RAG 仍能提升安全检测能力
2. **构建安全知识库** - 将风险识别规则、危机干预脚本、评估清单作为可检索的知识库
3. **利用对话历史** - 检索前 N 轮对话上下文，避免 LLM 遗漏关键信息

### 安全策略
1. **采用多类别风险分类** - 7 种风险类别覆盖主要场景
2. **安全优先原则** - 宁可误报不可漏报
3. **持续监控每轮对话** - 不仅在开始时检测，每轮都要评估

### 模型选择
- 如果成本敏感，可以用较小模型 + RAG 达到接近大模型的安全检测效果
- 大模型 + RAG 的组合效果最佳但边际收益递减
