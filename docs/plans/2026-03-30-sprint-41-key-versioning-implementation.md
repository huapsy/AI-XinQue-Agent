# Sprint 41 实施计划：密钥版本化最小闭环

## 背景

`Sprint 41` 的完整目标包含密钥轮换、租户边界、权限和审计基线，但当前代码缺口中最明确且可低风险落地的部分，是应用层加密的“版本化写入 + 历史兼容读取”。

## 本轮收敛范围

本轮只交付以下最小闭环：

- 为新密文增加密钥版本前缀
- 通过运行时配置声明当前写入版本
- 保持旧格式 `enc::...` 密文仍可解密
- 为后续多版本密钥共存保留稳定格式边界

## 不在本轮实现

- 多租户权限模型
- 企业管理侧 RBAC
- KMS 集成
- 真正的新旧密钥并行解密矩阵

## 实施步骤

1. 先补测试，明确新旧密文格式与配置读取契约
2. 在 `settings.py` 中增加加密版本配置读取
3. 在 `encryption_helpers.py` 中将新写入格式升级为版本化前缀
4. 保持 `decrypt_text()` 同时兼容旧格式与新格式
5. 回填 `Sprint 41` evaluation 与状态总览

## 预期结果

完成后，当前应用层加密将从“只有单密钥、无版本”升级为“有显式写入版本、保留历史兼容”的第一版可轮换边界。
