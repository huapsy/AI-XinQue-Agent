# RAG 知识库设计

> 来源：Gupta et al. (2025)、Limbic AI 专利

---

## 一、RAG 在心理健康 AI 中的作用

### 1.1 为什么需要 RAG

| 纯 LLM 的问题 | RAG 如何解决 |
|---------------|-------------|
| 可能产生幻觉（编造治疗方法） | 检索已验证的临床知识作为回应依据 |
| 缺乏具体的干预脚本 | 从知识库中检索匹配的干预步骤 |
| 安全检测能力不足（小模型尤其明显） | 检索相似案例辅助风险判断 |
| 不了解具体的危机干预协议 | 检索标准化的危机处理流程 |
| 无法保持跨会话的治疗一致性 | 检索患者历史信息和治疗计划 |
| 无法保证回应符合特定疗法流程 | 检索疗法的标准步骤和话术 |

### 1.2 实证数据（Gupta 论文）

**对安全意图检测的影响**：

| 模型 | 无RAG准确率 | 有RAG准确率 | 提升 | 统计显著性 |
|------|-----------|-----------|------|----------|
| GPT-4.1 nano | 48.3% | 72.7% | +24.4% | χ²≈256, p≪0.001 |
| o4-mini | 71.4% | 79.2% | +7.8% | χ²≈59, p≪0.001 |
| Gemini 2.5 flash | 78.1% | 82.0% | +3.9% | χ²=18（边界显著） |
| Gemini 2.5 Pro | 77.9% | 80.1% | +2.2% | 不显著 |
| Claude Sonnet 4 | 76.1% | 82.6% | +6.4% | χ²≈45, p≪0.001 |
| GPT-5 mini | 78.4% | 80.1% | +1.7% | 不显著 |

**分类别改善**（GPT-4.1 nano 为例）：

| 风险类别 | 无RAG召回率 | 有RAG召回率 | F1提升 |
|---------|-----------|-----------|--------|
| 虐待儿童 | 2.7% | 50.0% | 0.05→0.63 |
| 恐慌发作 | 16.3% | 73.6% | 0.28→0.78 |

**模型一致性提升**：

| 模型 | 平均模型共识度（有RAG） |
|------|---------------------|
| GPT-4.1 nano | 0.568（较无RAG提升15-25%） |
| o4-mini | 0.818 |
| Gemini 2.5 flash | 0.836 |
| Claude Sonnet 4 | 0.839 |
| GPT-5 mini | 0.863 |

**核心结论**：
- 小模型从 RAG 获益最大，大模型获益有限但知识准确性仍受益
- RAG 将错误从"漏检"转为"误报"——安全场景中这是更可接受的权衡
- RAG 让不同模型行为趋于一致，提供可预测性

## 二、知识库分类与内容设计

### 2.1 知识库总体结构

```
心雀知识库
  │
  ├── [A] 临床知识库 — 治疗方法、临床指南、心理学理论
  │
  ├── [B] 干预脚本库 — 具体练习和干预的执行步骤
  │
  ├── [C] 安全案例库 — 风险场景标注案例（用于安全分类增强）
  │
  ├── [D] 对话模板库 — 预定义的对话提示和回应模板
  │
  └── [E] 用户信息库 — 每个用户的历史画像和治疗进度
```

### 2.2 [A] 临床知识库

**条目格式**：

```yaml
id: CBT-CR-001
category: 认知行为治疗 > 认知重构
keywords: [认知扭曲, 自动思维, 认知重构, 苏格拉底式提问]
applicable_emotions: [焦虑, 抑郁, 愤怒, 自我否定]
applicable_distortions: [灾难化, 黑白思维, 读心术, 个人化]
difficulty: 中等
prerequisite_alliance: 中等以上
```

**条目内容结构**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 概述 | 这是什么技术/方法 | "认知重构是CBT核心技术，帮助识别和挑战不合理的自动思维" |
| 适用场景 | 什么时候使用 | "用户表达明显认知扭曲时" |
| 不适用场景 | 什么时候不用 | "用户处于危机状态；联盟未建立；痛苦有充分现实基础" |
| 标准步骤 | 分步操作指南 | "1.识别自动思维 2.情绪关联 3.检验证据 4.替代想法 5.评估变化" |
| 每步的话术示例 | 具体怎么说 | "当时你脑海中在想什么？" |
| 注意事项 | 常见陷阱和禁忌 | "不要强迫用户接受替代想法" |
| AI适配说明 | AI执行时的特殊考量 | "循序渐进引导，每步等用户回应" |

**需覆盖的临床领域**：

| 领域 | 技术/主题 |
|------|---------|
| 认知行为治疗(CBT) | 认知重构、行为激活、暴露层级、问题解决训练、苏格拉底式提问 |
| 正念技术 | 正念呼吸、身体扫描、正念观察想法、接纳与承诺、慈心冥想 |
| 情绪管理 | 情绪识别命名、情绪调节策略(TIPP)、痛苦承受技巧、情绪冲浪 |
| 人际关系 | "我"陈述沟通、边界设定、冲突解决、社交技能训练 |
| 睡眠健康 | 睡眠卫生教育、刺激控制、睡眠限制、失眠认知技术 |
| 压力管理 | 压力源识别、应对策略分类、时间管理、放松训练 |
| 焦虑应对 | 渐进式肌肉放松、焦虑阶梯暴露、担忧时间限制、不确定性承受 |
| 抑郁应对 | 行为激活计划、愉悦活动安排、社交重建、成就感记录 |
| 自我关怀 | 自我同情三要素、积极自我对话、自我照顾计划 |
| 创伤(有限) | 心理教育(正常化创伤反应)、稳定化技术、安全岛想象 — 深度处理必须转介 |

### 2.3 [B] 干预脚本库

每个脚本提供完整执行指南，包括引入话术、分步脚本、各种用户反应的处理方式。

**示例：4-7-8 呼吸法脚本**

```
id: INT-BREATH-001
适用情绪: 焦虑, 恐慌, 紧张, 压力, 愤怒
时长: 3-5分钟
难度: 简单
所需联盟: 低（紧急时可跳过联盟确认）

[引入]
"我知道一个简单的呼吸练习，可以帮你在感到{情绪}时平静下来。
只需要几分钟，我来一步步带你做。准备好了吗？"

[用户拒绝时]
"完全没关系。如果改变主意随时告诉我。我们继续聊。"

[步骤1 - 准备]
"找一个舒服的姿势。如果可以，轻轻闭上眼睛。准备好了告诉我。"

[步骤2 - 第一轮呼吸]
"用鼻子慢慢吸气，心里默数4个数。吸...2...3...4..."
"屏住呼吸，默数7个数。屏住...2...3...4...5...6...7..."
"用嘴巴慢慢呼气，默数8个数。呼...2...3...4...5...6...7...8..."

[步骤3 - 重复3-4轮]

[步骤4 - 结束]
"最后一轮...好的，可以慢慢睁开眼睛了。"

[反馈收集]
"你现在感觉怎么样？有什么变化吗？"

[用户反应处理]
- "好了一点" → "很高兴！以后感到紧张时随时可以用。"
- "没什么感觉" → "没关系，多练几次才会有明显感觉。睡前试试可能更容易感受到。"
- "做不到" → "没问题，不是每种方法都适合。你觉得难在哪？我们可以调整或换别的。"
- "更紧张了" → "谢谢你告诉我。刚开始可能会更关注身体反而紧张。我们先停下来聊聊。"
```

**需准备的干预脚本清单**：

| 类型 | 脚本 | 难度 | 所需联盟 |
|------|-----|------|---------|
| 呼吸练习 | 4-7-8呼吸法、腹式呼吸、方块呼吸 | 简单 | 低 |
| 肌肉放松 | 渐进式肌肉放松（全身/简化版） | 简单 | 低 |
| 正念 | 正念呼吸(3分钟)、五感练习、身体扫描 | 中等 | 低-中 |
| 认知重构 | 自动思维记录、证据检验、替代想法 | 中等 | 中 |
| 行为激活 | 愉悦活动清单、活动安排、成就感记录 | 中等 | 中 |
| 思维记录 | 三栏思维记录、情境-想法-情绪链 | 中高 | 中 |
| 情绪调节 | 情绪命名、TIPP技巧、安全岛想象 | 中等 | 低-中 |
| 自我关怀 | 自我同情信、善待自己练习 | 中等 | 中 |

### 2.4 [C] 安全案例库

用于 RAG 增强的安全分类。每条案例包含：原始文本、风险类别、风险等级、关键特征、正确分类、应采取行动。

**特别重要：包含"无风险但易混淆"的反例**，减少误报。

| 风险类别 | 建议最少案例数 | 说明 |
|---------|-------------|------|
| 自杀意念 | 50+ | 直接/间接/隐晦表达 |
| 自我伤害 | 30+ | 各种形式 |
| 恐慌发作 | 30+ | 各种生理/心理症状 |
| 虐待相关 | 40+ | 儿童虐待/家暴/性虐待 |
| 暴力威胁 | 20+ | 对他人的暴力意图 |
| 创伤披露 | 30+ | 各类创伤事件 |
| 无风险(易混淆) | 50+ | "累死了"≠自杀、"气死了"≠暴力 |
| 提示注入 | 20+ | 各类攻击模式 |

### 2.5 [D] 对话模板库

按场景分类存储模板：共情回应、开场问候、联盟确认、不对齐恢复（每种类型）、危机回应、转介、干预邀请、会话结束。

详见 `02-对话对齐与治疗联盟机制.md` 中的模板部分。

### 2.6 [E] 用户信息库

每用户维护结构化 JSON 档案，包含：
- 基本画像（称呼、年龄段、沟通风格偏好）
- 临床模式（认知扭曲、情绪模式、行为模式）
- 治疗进度（联盟分数、会话数、干预历史及用户反馈）
- 安全记录（风险等级、风险历史）
- 偏好记忆（喜欢/不喜欢的方式、个人化细节）

## 三、检索机制设计

### 3.1 检索架构

```
用户输入文本
  │
  ▼
[嵌入模型] → 查询向量
  │
  ▼ (多路并行检索)
  ├── 临床知识库 → Top-3
  ├── 安全案例库 → Top-5（含正反例）
  ├── 干预脚本库 → Top-2（仅干预时）
  ├── 对话模板库 → Top-1（按对话阶段过滤）
  └── 用户信息库 → 精确ID匹配
  │
  ▼
[结果组装] → 注入系统提示
```

### 3.2 混合检索策略

```
final_score = 0.7 × 语义相似度(余弦) + 0.3 × 关键词相似度(BM25)
```

### 3.3 检索参数

| 参数 | 建议值 |
|------|--------|
| Top-K 临床知识 | 3-5（Limbic专利建议K=5） |
| Top-K 安全案例 | 5-8 |
| 相似度阈值 | 0.70 |
| 最大检索内容 | 2000 tokens |

### 3.4 在系统提示中的位置

```
[1] 角色定义
[2] 行为准则
[3] 用户画像摘要（来自用户信息库）
[4] 当前治疗目标和阶段
[5] 临床知识检索结果 ← RAG
[6] 安全案例参考 ← RAG（安全检测时）
[7] 干预脚本 ← RAG（干预执行时）
[8] 对话历史（最近N轮）
[9] 用户当前输入
```

## 四、Azure 实现方案

| 组件 | Azure 服务 |
|------|-----------|
| 向量存储与检索 | Azure AI Search（向量索引） |
| 嵌入生成 | Azure OpenAI Embeddings API |
| 结构化数据存储 | Azure Cosmos DB |
| 知识库内容管理 | Azure Blob Storage + 版本控制 |
| 全文搜索(BM25) | Azure AI Search |

## 五、维护策略

- 临床知识：每季度由专业人员审核更新
- 干预脚本：根据用户反馈持续优化
- 安全案例：每月从实际对话补充（特别是漏检案例）
- 用户信息：每次会话后自动更新
- 所有内容变更需版本控制和变更日志
